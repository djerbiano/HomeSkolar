# 🎓 HomeSkolar – Conception d’une plateforme de soutien scolaire à distance

## 📚 Contexte du projet

HomeSkolar est une association qui met en relation des enfants en difficulté scolaire avec des tuteurs bénévoles.  
L’objectif est de concevoir une solution web permettant :

- L’assignation d’un tuteur à chaque élève
- La planification de rendez-vous hebdomadaires
- Le suivi des tâches post-rencontre
- La communication simplifiée entre élèves et tuteurs

Ce projet se concentre sur la **conception technique** et la **planification**, sans phase de développement ni déploiement.

---

## 🎯 Objectifs pédagogiques

- ✅ Concevoir une solution technique répondant aux besoins d’un client
- ✅ Définir les exigences fonctionnelles à partir du besoin client
- ✅ Gérer un projet avec les méthodologies agiles
- ✅ Mener une veille technologique
- ✅ Rédiger un cahier des charges complet
- ✅ Produire un diagramme de classes UML
- ✅ Élaborer un backlog produit avec user stories priorisées et estimées

---

## 📚 Stack technique retenue (conception)

| Composant | Technologie choisie | Justification |
|-----------|---------------------|----------------|
| Front-end | React | Composants réutilisables, interface dynamique (calendrier, messagerie) |
| Back-end | Java + Spring Boot | Orienté objet, aligné UML, sécurité intégrée (Spring Security + JWT) |
| Base de données | MongoDB | Flexibilité du schéma, adapté aux évolutions futures |
| Authentification | JWT | Stateless, adapté aux API REST |

### Veille technologique

- **Outil** : Feedly (centralisation des flux RSS)
- **Sources suivies** : React, Spring Boot, MongoDB
- **Objectif** : rester informé des mises à jour, bonnes pratiques et évolutions de sécurité

---

## ✨ Fonctionnalités conçues

| Domaine | Fonctionnalités |
|---------|----------------|
| 🔐 Authentification | Inscription, connexion, gestion/réinitialisation mot de passe, rôles (élève/tuteur/admin) |
| 👥 Gestion utilisateurs | Assignation élève → tuteur (admin), activation/désactivation, suppression logique |
| 💬 Messagerie | Échange élève ↔ tuteur, épinglage de messages, notifications non lus, historique |
| 📅 Calendrier | Vues jour/semaine/mois, visualisation des rendez-vous |
| 📆 Rendez-vous | Définition des disponibilités (tuteur), réservation, annulation, report |
| ✅ Tâches | Création post-rencontre, tâches personnelles (mémos), suivi, notifications de rappel |

## 🧑‍💻 Organisation de l’équipe (hypothèse de développement)

- **4 développeurs** : 2 back-end, 2 front-end
- **Méthodologie** : Agile (Sprints)
- **Estimation totale** : ~165 heures de développement

### Priorisation des sprints (prévisionnelle)

| Sprint | Périmètre | Durée estimée |
|--------|-----------|----------------|
| Sprint 1 | Authentification + Gestion utilisateurs | ~1 semaine |
| Sprint 2 | Disponibilités + Rendez-vous + Calendrier | ~1 semaine |
| Sprint 3 | Tâches + Notifications | ~1 semaine |
| Sprint 4 | Messagerie + Finalisations | ~1 semaine |
